# HOZO 總控專案分類說明

這份文件用來定義 HOZO 總控任務、LINE 訊息判斷、每日報告與權責指派時，「總控專案」欄位應該如何分類。

使用原則：

- 優先選擇最接近實際責任線的總控專案。
- 不要只因為訊息來自某個 LINE 群組就直接套用分類，仍要看事件內容。
- 如果同一件事同時跨多個分類，先選擇「主要要被追蹤與驗收」的那一類。
- 財務、客人服務、工程、工務、房務應盡量獨立，不要全部塞進公司治理或營運交接。

## 建議放置位置

使用者最容易看到的位置應該是 Notion，而不是只放在 GitHub 或本機專案。

建議放置：

1. Notion：放在 HOZO 總控任務庫或總控資料庫區塊旁，命名為「HOZO 總控專案分類說明」。
2. Notion database：若 `總控專案` 是 select 欄位，將上述名稱設為選項，並在資料庫說明或旁邊說明頁連到這份分類表。
3. 專案文件：保留本檔作為工程與版本管理來源。
