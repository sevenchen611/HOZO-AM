# UPGRADE-2026-06-13-AM-IMP-2026.0613.02

Project: HOZO AM

Version: `AM-IMP-2026.0613.02`

## Summary

Worker wrapper OS-level log redirection: `start-local-worker.ps1` must not pipe node output through PowerShell (`Tee-Object` or pwsh `>`/`>>` operators). When that object pipeline stalls, node's stdout pipe fills, console.log blocks, and the worker freezes silently — discovered in this project during the Codex backend rollout (worker stuck for 8+ minutes with no self-test, no timeout, no children). The wrapper now uses `cmd /c "node ... >> log 2>&1"` so the OS owns the file append.

## Changes

- `scripts/start-local-worker.ps1`: cmd /c append redirection replaces the Tee-Object pipeline; pitfall documented in the header comment. Same fix applied to SevenAM.

## Environment Variables (names only)

None.

Launcher script only.

## Verification

Under the new wrapper the codex self-test logs within ~17 seconds and the operating-hours gate engages; under Tee-Object the identical worker produced no output indefinitely. Diagnosis isolated the wrapper by running the worker directly with `Start-Process -RedirectStandardOutput` (worked) versus through the pwsh pipeline (froze).
