Target project: HOZO AM

Owner: HOZO AM / Codex

## Purpose

This improvement makes every scheduled report slot ask for a clear completion goal before a task or project is treated as trackable progress. It solves the problem where projects and tasks can look active but still lack a responsible owner's definition of done.

The required workflow is:

1. The responsible owner gives an oral explanation of the goal.
2. The oral explanation is entered into the tracking table as `目標口述原文`.
3. Codex confirms whether the goal is specific enough to track.
4. Only after Codex confirmation does the next report slot tell the owner what to do next and update progress.
5. LINE or Email can be used to ask for missing goal definitions.
